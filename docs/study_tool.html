📚 Study Summary

Project Overview

This project is a single-file Pygame simulation where 50 square agents move with angle-based velocity, avoid larger neighbors, chase smaller neighbors, bounce at boundaries, and respawn after lifespan expiry.

Core Concepts Covered

• Source Evidence Order: Python files scanned deterministically as gitignore.py, main.py, tempCodeRunnerFile.py; only main.py contains executable project logic.

• Data Flow: main() orchestrates event polling, per-square updates, drawing, replacement, and HUD rendering each frame.

• Behavior Logic: Square.update() combines random direction drift, flee vectors, chase vectors, and wall reflection.

• State Mutation: Position (x, y), angle, and age are mutated every frame; lifespan replacement keeps population constant.

• Complexity: Neighbor scans in compute_flee_vector and compute_chase_vector imply per-frame interaction cost proportional to $O(n^2)$ for n squares.

• Robustness Insight: Age is incremented both inside Square.update() and again in the lifecycle loop, which accelerates expiration relative to a single increment design.
